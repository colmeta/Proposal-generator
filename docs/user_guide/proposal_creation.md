# Proposal Creation Guide

This guide provides detailed instructions for creating proposals using the Proposal Generator.

## Overview

The proposal creation process involves several steps:
1. Basic Information
2. Project Details
3. Team Information
4. Budget Planning
5. Supporting Documents
6. Review and Submit

## Step-by-Step Process

### Step 1: Basic Information

#### Proposal Title
- **Required**: Yes
- **Guidelines**:
  - Be clear and descriptive
  - Include key project elements
  - Keep it concise (50-100 characters recommended)
  - Example: "AI-Powered Climate Change Prediction System"

#### Funder Information
- **Funder Name**: Required
  - Enter the full name of the funding organization
  - Example: "National Science Foundation"
  
- **Funder Website**: Optional
  - URL of the funding organization
  - Used for research and guidelines extraction

#### Proposal Type
- **Required**: Yes
- **Options**:
  - Research Grant
  - Project Funding
  - Scholarship
  - Fellowship
  - Other

#### Submission Deadline
- **Required**: Yes
- **Format**: Date picker
- **Tip**: Set reminders well before the deadline

#### Budget
- **Required**: Yes
- **Format**: Enter as number or with currency symbol
  - Examples: `50000`, `$50,000`, `â‚¬45,000`
- **Tip**: Be realistic and justified

### Step 2: Project Information

#### Project Description
- **Required**: Yes
- **Length**: 500-2000 words recommended
- **Should Include**:
  - Problem statement
  - Project goals
  - Significance and impact
  - Innovation and novelty

#### Objectives
- **Required**: Yes
- **Format**: Bullet points or numbered list
- **Guidelines**:
  - Be specific and measurable
  - Use action verbs
  - Align with funder priorities
  - Example:
    - Develop a novel AI algorithm for climate prediction
    - Validate the algorithm using historical data
    - Publish findings in peer-reviewed journals

#### Methodology
- **Required**: No (but highly recommended)
- **Should Include**:
  - Research approach
  - Data collection methods
  - Analysis techniques
  - Timeline and milestones

#### Expected Outcomes
- **Required**: No
- **Should Include**:
  - Deliverables
  - Publications
  - Impact metrics
  - Long-term benefits

### Step 3: Team Information

#### Principal Investigator (PI)
- **Required**: Yes
- **Information Needed**:
  - Full name
  - Email address
  - Affiliation/Institution
  - Brief bio (optional)

#### Co-Investigators
- **Required**: No
- **Format**: One per line
- **Include**:
  - Name
  - Affiliation
  - Role in project

#### Team Members
- **Required**: No
- **Format**: One per line
- **Include**:
  - Name
  - Role
  - Contribution

### Step 4: Budget Breakdown

#### Adding Budget Items
1. Click "Add Budget Item"
2. Enter category (e.g., Personnel, Equipment, Travel)
3. Enter amount
4. Add description (optional)
5. Repeat for all categories

#### Common Categories
- **Personnel**: Salaries, benefits
- **Equipment**: Hardware, software
- **Travel**: Conferences, field work
- **Materials**: Supplies, consumables
- **Indirect Costs**: Overhead, facilities

#### Tips
- Align with funder guidelines
- Justify each expense
- Include contingency if allowed
- Check for maximum limits per category

### Step 5: Additional Information

#### Keywords
- **Format**: Comma-separated
- **Purpose**: Help with categorization and search
- **Example**: `AI, climate change, machine learning, prediction`

#### References
- **Format**: Free text or formatted citations
- **Include**: Key papers, reports, guidelines

#### Internal Notes
- **Private**: Not included in generated proposal
- **Use for**: Reminders, internal comments, revision notes

### Step 6: Supporting Documents

#### Supported File Types
- PDF (.pdf)
- Word documents (.docx, .doc)
- Text files (.txt)

#### Common Documents
- CVs/Resumes
- Letters of support
- Previous proposals
- Publications
- Budget justifications

#### Tips
- Keep file sizes reasonable (<10MB each)
- Use descriptive filenames
- Ensure documents are relevant

### Step 7: Review and Submit

#### Preview
- Click "Preview" to review your proposal
- Check all sections for completeness
- Verify formatting and content

#### Save as Draft
- Use "Save as Draft" to save progress
- Drafts can be edited later
- Not submitted for generation

#### Submit
- Click "Submit Proposal" when ready
- This creates a job for proposal generation
- Monitor progress in the Jobs dashboard

## Best Practices

### Content Quality
- **Be Specific**: Avoid vague statements
- **Be Concise**: Respect word limits
- **Be Clear**: Use simple, direct language
- **Be Compelling**: Highlight impact and innovation

### Alignment
- **Match Funder Priorities**: Align with funder goals
- **Follow Guidelines**: Adhere to format requirements
- **Address Criteria**: Cover all evaluation criteria

### Review Process
- **Self-Review**: Check for errors and clarity
- **Peer Review**: Get feedback from colleagues
- **Expert Review**: Consult with experienced grant writers

### Common Mistakes to Avoid
- Missing required fields
- Exceeding word limits
- Unclear objectives
- Unjustified budget
- Poor formatting
- Missing deadlines

## After Submission

### Job Monitoring
1. Navigate to Jobs dashboard
2. Monitor real-time progress
3. Check for errors or warnings
4. Review task completion

### Document Review
1. Once complete, view generated document
2. Review all sections
3. Check formatting
4. Verify content accuracy

### Revisions
1. If needed, create a new version
2. Make necessary changes
3. Resubmit for regeneration
4. Compare versions

## Tips for Success

1. **Start Early**: Give yourself plenty of time
2. **Plan Ahead**: Outline before writing
3. **Be Thorough**: Complete all sections
4. **Review Carefully**: Check for errors
5. **Seek Feedback**: Get input from others
6. **Iterate**: Improve based on feedback

## Troubleshooting

### Common Issues
- **Form Validation Errors**: Check required fields
- **Budget Format**: Use numbers or currency format
- **File Upload Issues**: Check file size and type
- **API Errors**: Verify API connection

For more help, see the [Troubleshooting Guide](troubleshooting.md).
